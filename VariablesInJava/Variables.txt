Java Variables
A variable is a named memory location used to store data.

Every variable in Java must have:
1. A data type
2. A unique name
3. A value (optional at declaration)

# Creating Variables in Java:

int problemsSolved = 120;

int → data type
problemsSolved → variable name
120 → value stored
This means we are storing the value 120 inside a memory location named problemsSolved.

# Declaration and Assignment

double completionRate;
completionRate = 87.5;
Or do both together:

boolean isPremiumMember = true;
Both approaches are correct.

Java is Statically Typed
Java requires that every variable be declared before it is used.

This is valid:
int totalContests = 15;

This is invalid:
totalContests = 15;  // ❌ Not allowed

Java must know the data type first.

# Changing Variable Values
The value stored inside a variable can change.

int rank = 450;
// after next contest
rank = 320;
The memory location stays the same, but the value updates.

That’s why it is called a variable.

# Important Rule: Type Cannot Change
Once declared, a variable cannot change its data type.

This is invalid:
int streak = 30;
double streak = 30.5;   // ❌ Not allowed
Java does not allow redeclaration with a different type in the same scope.

# Naming Rules for Variables
-> Can contain letters, digits, _, $
-> Cannot start with a digit
-> Cannot use Java keywords
-> Case-sensitive

Example:

int score;
int Score;  // different variable

# Java Literals
A literal is the actual fixed value assigned to a variable.

Example:

int activeUsers = 5000;
Here:

activeUsers → variable
5000 → literal
Java supports multiple types of literals.

Integer Literals
Integer literals represent whole numbers.

# Java supports 4 formats:

1️⃣ Decimal (Base 10)
int enrolledStudents = 250;

2️⃣ Binary (Base 2)
int accessMask = 0b10101;
0b indicates binary.

3️⃣ Octal (Base 8)
int filePermission = 075;
Leading 0 indicates octal.

4️⃣ Hexadecimal (Base 16)
int themeColor = 0xFF;
0x indicates hexadecimal.

Integer literals are used with:

byte, short, int, long.

Floating-Point Literals
Floating-point literals represent decimal numbers.

Example:

double averageScore = 78.6;
float discountPercent = 15.5F;
By default:

Decimal numbers are treated as double.

To store in float, you must use F.

# Scientific Notation

double serverLoad = 2.5e3;
This means:

2.5 × 10³ = 2500

This format is useful in data analytics and performance calculations.

Character Literals
Character literals represent a single character enclosed in single quotes.

char grade = 'A';
You can also use escape sequences:

char newLine = '\n';
char tabSpace = '\t';
Escape characters represent special formatting.

String Literals
A string literal is a sequence of characters inside double quotes.

String platformName = "CodeHelp ONE";
String courseTrack = "DSA + Core + Aptitude";
Anything inside double quotes is treated as text.

Boolean Literals
Boolean literals represent logical values.

They can only be:

boolean isLoggedIn = true;
boolean hasCompletedSheet = false;
Only true and false are valid.
